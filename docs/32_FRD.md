#### 6.3 发布调度详细设计

##### 6.3.1 业务流程

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│  选择内容   │ -> │  选择平台   │ -> │  设置时间   │ -> │  确认发布   │
│             │    │             │    │             │    │             │
└─────────────┘    └─────────────┘    └─────────────┘    └─────────────┘
                                                          │
                                          ┌───────────────┘
                                          ▼
                                   ┌─────────────┐
                                   │ 加入发布队列 │
                                   │  等待执行   │
                                   └─────────────┘
                                          │
                    ┌─────────────────────┼─────────────────────┐
                    ▼                     ▼                     ▼
             ┌─────────────┐       ┌─────────────┐       ┌─────────────┐
             │  到达时间   │       │  调用平台API │       │  返回结果   │
             │  触发执行   │  ->   │  发布内容   │  ->   │  记录状态   │
             └─────────────┘       └─────────────┘       └─────────────┘
```

##### 6.3.2 调度算法

| 特性 | 说明 |
|------|------|
| 触发方式 | 定时任务轮询（每分钟检查） |
| 执行方式 | 异步执行，不阻塞主线程 |
| 重试机制 | 失败后3次重试，间隔1分钟 |
| 并发控制 | 同一账户最多3个并发发布 |

##### 6.3.3 页面设计

| 页面 | 元素 | 说明 |
|------|------|------|
| 调度列表页 | 内容标题、目标平台、发布时间、状态 | 显示所有调度任务 |
| 创建调度弹窗 | 内容选择、平台多选、时间选择器 | 创建新的调度任务 |

##### 6.3.4 API接口

| 接口 | 方法 | 说明 |
|------|------|------|
| /api/schedules | GET | 获取调度列表 |
| /api/schedules | POST | 创建调度 |
| /api/schedules/{id} | GET | 获取调度详情 |
| /api/schedules/{id} | PUT | 更新调度 |
| /api/schedules/{id} | DELETE | 删除调度 |
| /api/schedules/{id}/cancel | POST | 取消调度 |

##### 6.3.5 数据模型

```
Schedule:
- id: UUID (主键)
- content_id: UUID (外键)
- social_account_id: UUID (外键)
- scheduled_at: datetime
- published_at: datetime (可选)
- status: string (pending/published/failed)
- platform_post_id: string (可选)
- error_message: string (可选)
- created_at: datetime
```

##### 6.3.6 状态流转

```
                    ┌─────────────┐
                    │   pending   │
                    │    (待发布)  │
                    └──────┬──────┘
                           │
            ┌──────────────┼──────────────┐
            │              │              │
            ▼              ▼              ▼
     ┌─────────────┐ ┌─────────────┐ ┌─────────────┐
     │  published  │ │   failed    │ │  cancelled  │
     │   (已发布)  │ │  (发布失败)  │ │  (已取消)   │
     └─────────────┘ └─────────────┘ └─────────────┘
```

```
Analytics:

- platform: string

```

##### 7.3.5 数据更新策略

| 数据类型 | 更新频率 | 更新方式 |
|----------|---------|---------|
